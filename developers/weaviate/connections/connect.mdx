:::warning Use authentication
By default Weaviate allows unauthenticated connections from `localhost`.

Do not expose your Weaviate instance to the public internet if authentication is not enabled. If you do not enable authentication, anyone who has the URL can access your instance.
:::

## Authentication enabled

If [authentication](/developers/weaviate/configuration/authentication.md) is enabled, use a Weaviate API key or OIDC credentials to connect.

#### Weaviate API keys

To authenticate with a Weaviate API key, follow these examples.

Be sure to use your Weaviate API key to authenticate and not the API key for a third-party service like OpenAI.

OIDC providers use a variety of authentication configurations. The example shows a typical OIDC configuration, but your identity provider may use a different configuration. For details, see [OIDC authentication](../configuration/authentication.md#oidc---a-systems-perspective).

## Local connection URL

Docker instances default to `http://localhost:8080`.

If your instance runs on Kubernetes, see the `host` and `port` values in your Helm chart's `values.yaml` file.
